# Подробный промт реализации: Система регистрации с QR-кодами

## Технологический стек

- **Laravel 13**, **Filament v5.6.1**, **Livewire**
- **simplesoftwareio/simple-qrcode** (уже установлена)
- **maatwebsite/excel** (установить)
- SQLite (текущая БД)
- Минимальный стиль публичных страниц

---

## Порядок реализации

---

### Этап 1: Установка зависимостей и конфигурация

**1.1** Установить пакет `maatwebsite/excel` через Composer.

**1.2** Добавить в `.env` переменную `YANDEX_WEBHOOK_SECRET` со случайным секретным ключом.

**1.3** Добавить в `.env.example` ту же переменную с placeholder-значением.

**1.4** В `config/services.php` добавить секцию `webhook` с ключом `yandex_secret`, читающим значение из окружения.

---

### Этап 2: Миграции базы данных

**2.1** Создать миграцию `add_registration_fields_to_events_table`:
- Добавить поле `capacity` — integer, nullable (макс. количество мест)
- Добавить поле `registration_deadline` — datetime, nullable (крайний срок регистрации)
- Добавить поле `questions` — json, nullable (кастомные вопросы формы)
- Добавить поле `yandex_form_id` — string(100), nullable (ID формы в Яндексе)

**2.2** Создать миграцию `create_participants_table`:
- `id` — bigint, PK
- `event_id` — bigint, FK на `events.id`, cascade delete
- `name` — string(255)
- `email` — string(255), nullable
- `phone` — string(20), nullable
- `answers` — json (все ответы из формы)
- `checkin_token` — string(64), unique, nullable
- `checked_in_at` — timestamp, nullable
- `status` — string(20), default `registered` (варианты: registered, verified, arrived, cancelled)
- `ticket_sent_at` — timestamp, nullable
- `verification_code` — string(6), nullable
- `verification_code_sent_at` — timestamp, nullable
- `email_verified_at` — timestamp, nullable
- `source` — string(50) (yandex_form, manual, import)
- `utm_tags` — json, nullable
- `created_at`, `updated_at` — timestamps

Индексы:
- `participants_email_index` по полю `email`
- `participants_checkin_token_unique` уникальный по `checkin_token`
- `participants_event_id_status_index` составной по `event_id` + `status`

---

### Этап 3: Модели

**3.1** Обновить модель `Event` (`app/Models/Event.php`):
- Добавить новые поля в `$fillable`: `capacity`, `registration_deadline`, `questions`, `yandex_form_id`
- Добавить связь `participants()` — `hasMany(Participant::class)`

**3.2** Создать модель `Participant` (`app/Models/Participant.php`):
- `$fillable` — все поля миграции
- `$casts` — `answers` → array, `utm_tags` → array, `checked_in_at` → datetime, `ticket_sent_at` → datetime, `verification_code_sent_at` → datetime, `email_verified_at` → datetime
- Связь `event()` — `belongsTo(Event::class)`
- Метод `generateCheckinToken()` — генерирует уникальный токен длиной 40 символов через `Str::random(40)`, сохраняет в `checkin_token`
- Метод `generateVerificationCode()` — генерирует 6-значный код через `random_int(100000, 999999)`, сохраняет в `verification_code` и `verification_code_sent_at`
- Метод `getStatusLabelAttribute()` — возвращает человекочитаемый статус
- Метод `getStatusColorAttribute()` — возвращает цвет для Filament-бейджа (registered → gray, verified → blue, arrived → green, cancelled → red)
- Метод `scopeForEvent($query, $eventId)` — фильтрация по мероприятию

### Этап 4: API Webhook для Яндекс.Форм

**4.1** Создать файл `routes/api.php`:
- Маршрут `POST /yandex/register` → `YandexWebhookController@handle`

**4.2** Подключить API маршруты в `bootstrap/app.php`:
- Добавить `->withRouting(api: __DIR__.'/../routes/api.php', apiPrefix: 'api')`

**4.3** Создать контроллер `app/Http/Controllers/Api/YandexWebhookController.php`:
- Метод `handle(Request $request)`
- Проверить заголовок `X-Yandex-Webhook-Secret` на соответствие значения из `config('services.webhook.yandex_secret')`
- Если заголовок не совпадает — вернуть 403
- Извлечь `event_id` из тела запроса
- Проверить существование мероприятия (и что оно опубликовано)
- Извлечь `name`, `email`, `phone` из данных формы
- Проверить дубликаты: существует ли участник с таким `email` + `event_id`
- Если дубликат — вернуть 200 с сообщением «уже зарегистрирован» (не ошибку, чтобы Яндекс не повторял)
- Создать запись в `participants` с автоматической генерацией `checkin_token`
- Сохранить все ответы из формы в поле `answers` (JSON)
- Установить `source` = `yandex_form`
- Логировать создание участника
- Вернуть 200 OK

### Этап 5: Публичные страницы — маршруты

**5.1** Добавить в `routes/web.php` маршруты:

- `GET /ticket/{token}` → `TicketController@show` (имя: `ticket.show`)
- `GET /checkin/{token}` → `CheckinController@handle` (имя: `checkin.handle`), middleware `auth`
- `GET /recovery` → `RecoveryController@showForm` (имя: `recovery.form`)
- `POST /recovery` → `RecoveryController@sendCode` (имя: `recovery.send`)
- `GET /recovery/code` → `RecoveryController@showCodeForm` (имя: `recovery.code.form`)
- `POST /recovery/code` → `RecoveryController@verifyCode` (имя: `recovery.code.verify`)

---

### Этап 6: Публичные страницы — контроллеры

**6.1** Создать `app/Http/Controllers/TicketController.php`:
- Метод `show($token)` — найти участника по `checkin_token`, если не найден — 404
- Сгенерировать QR-код через `QrCode::format('svg')->size(300)->generate(url("/checkin/{$token}"))`
- Передать в Blade: participant, event, qrcode (SVG string), ticketUrl
- Отобразить шаблон `ticket.show`

**6.2** Создать `app/Http/Controllers/CheckinController.php`:
- Метод `handle($token)` — доступен только авторизованным пользователям (middleware `auth`)
- Найти участника по `checkin_token`, если не найден — 404
- Если `checked_in_at` уже заполнен — показать сообщение «Уже отмечен в {время}»
- Если не отмечен — установить `checked_in_at = now()`, `status = 'arrived'`, сохранить
- Показать сообщение «Успешно отмечен: {имя участника}»
- Отобразить шаблон `checkin.result`

**6.3** Создать `app/Http/Controllers/RecoveryController.php`:
- Метод `showForm()` — показать форму ввода email
- Метод `sendCode(Request $request)`:
  - Валидация email
  - Rate limit: 3 запроса с одного IP за 10 минут (через Cache)
  - Найти участника по email
  - Если не найден — показать сообщение (не раскрывать существование)
  - Сгенерировать 6-значный код через `generateVerificationCode()`
  - Отправить `VerificationCodeMail`
  - Сохранить email в session
  - Редирект на страницу ввода кода
- Метод `showCodeForm()` — показать форму ввода 6-значного кода
- Метод `verifyCode(Request $request)`:
  - Валидация кода (6 цифр)
  - Rate limit: 5 попыток с одного IP за 10 минут (через Cache)
  - Найти участника по email из session
  - Проверить: код совпадает И `verification_code_sent_at` не старше 15 минут
  - Если код верный — очистить `verification_code`, отправить `TicketMail`, редирект на страницу билета
  - Если неверный — показать ошибку

## Зависимость задач

```
Этап 1 (зависимости) → Этап 2 (миграции) → Этап 3 (модели)
                                              ↓
Этап 4 (API webhook) ← Этап 3
                      ↓
Этап 5 (маршруты) → Этап 6 (контроллеры) → Этап 7 (шаблоны)
                   → Этап 8 (письма)
                                              ↓
Этап 9 (EventResource) → Этап 10 (ParticipantResource) → Этап 11 (виджеты)
                                                          ↓
Этап 12 (экспорт) ← Этап 10
                   ↓
Этап 13 (команды) ← Этап 8
```

## Проверка после реализации

1. `php artisan migrate` — все миграции прошли
2. `php artisan tinker` — Participant::factory()->create() создаёт запись с токеном
3. POST-запрос на `/api/yandex/register` с валидным секретом — создаёт участника
4. POST-запрос с невалидным секретом — возвращает 403
5. POST-запрос с дублирующим email — возвращает 200 без создания
6. Открыть `/ticket/{token}` — билет с QR-кодом отображается
7. Открыть `/checkin/{token}` (будучи авторизованным админом) — участник отмечается
8. Открыть `/checkin/{token}` (повторно) — показывает «уже отмечен»
9. Открыть `/checkin/{token}` (без авторизации) — редирект на логин
10. Процесс восстановления: ввод email → получение кода → ввод кода → получение билета
11. Rate limit: 4-й запрос кода с IP — ошибка
12. Rate limit: 6-я попытка ввода кода — ошибка
13. Filament: список участников с фильтрами и бейджами
14. Filament: массовая отправка билетов
15. Filament: экспорт в Excel
16. `php artisan tickets:send {event_id}` — письма отправлены, `ticket_sent_at` обновлён
17. `php artisan import:yandex-csv file.csv` — импорт прошёл
