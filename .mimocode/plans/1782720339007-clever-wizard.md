# Оптимизация производительности страниц

## Найденные проблемы

### HIGH
1. **JS-бандл ~325KB** — Swiper (~150KB) и GSAP (~95KB) загружаются на все страницы через `app.js`, хотя нужны только на главной
2. **Дублирование `@vite`** — `app.js` подключается дважды в `home.blade.php` (head + body)
3. **4-запросный паттерн** поиска Event скопирован 4 раза в `routes/web.php`
4. **N+1 в Livewire** — EventHero, EventDocuments, Testimonials делают повторные запросы, данные уже загружены

### MEDIUM
5. **Нет lazy loading** у спикеров, гостей, постера, медиа-фото
6. **service worker** содержит 11 `console.log()`

---

## План исправлений

### 1. Хелпер `resolveActiveEvent()` — убрать дублирование в routes
**Файл:** `app/Helpers/helpers.php`

Создать функцию `resolveActiveEvent()` с одним `with()` массивом. Заменить 4 копии паттерна в `routes/web.php` на вызов хелпера.

### 2. Lazy loading для изображений
**Файлы:**
- `home.blade.php` — постер (строка ~122), медиа-фото (~137), лого в футере (~291)
- `livewire/event-speakers.blade.php` — фото спикеров
- `livewire/event-keynote-speakers.blade.php` — фото гостей
- `livewire/testimonials.blade.php` — фото отзывов

Добавить `loading="lazy" decoding="async" width="..." height="..."` ко всем ниже-фолд изображениям.

### 3. Убрать дублирование `app.js` из `home.blade.php`
**Файл:** `home.blade.php`

Удалить `resources/js/app.js` из `@vite` на строке ~374 (он уже подключён в head на строке 20).

### 4. Оптимизация Swiper CSS
**Файл:** `resources/css/app.css`

Заменить `@import 'swiper/swiper-bundle.css'` на импорт только нужных модулей:
```css
@import 'swiper/css';
@import 'swiper/css/pagination';
@import 'swiper/css/navigation';
```

### 5. Очистка console.log в service worker
**Файл:** `public/sw.js`

Удалить все `console.log()` вызовы.

### 6. Code-split через Vite manualChunks
**Файл:** `vite.config.js`

Добавить `build.rollupOptions.output.manualChunks` для выноса vendor-библиотек в отдельные чанки:
- `swiper` — отдельный чанк
- `gsap` — отдельный чанк
