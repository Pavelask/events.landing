# Исправление 403 ошибки при входе в Filament админку

## Проблема
После ввода логина и пароля на хостинге выдаётся 403 ошибка. Локально работает нормально.

## Вероятные причины

### 1. Не выполнены миграции на хостинге
Последние 5 миграций добавляют новые колонки в таблицу `events`:
- `privacy_cookie_banner_text`
- `show_cookie_banner`
- `privacy_cookie_banner_title`
- `privacy_cookie_policy`
- `show_personal_data_consent`

Если миграции не выполнены, Filament может падать при попытке загрузить Dashboard/ресурсы.

### 2. Кэш маршрутов/конфигов устарел
После добавления новых маршрутов (`cookie.policy`) и изменений в коде нужно пересобрать кэш.

### 3. Нет файла `composer.lock` или `vendor` на хостинге
Если `composer install` не был выполнен — Intervention Image и другие зависимости отсутствуют.

## План исправления

### Шаг 1: Выполнить на хостинге команды
```bash
composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

### Шаг 2: Если 403 сохраняется — добавить `canAccess` на Dashboard
В `AdminPanelProvider.php` добавить проверку доступа:
```php
->canAccess(fn () => auth()->check())
```

### Шаг 3: Если не помогает — временно отключить авторизацию панели для отладки
Добавить в `AdminPanelProvider.php`:
```php
->auth(false)
```
Для проверки, что проблема именно в авторизации.

## Файлы для изменения
- `app/Providers/Filament/AdminPanelProvider.php` (шаг 2-3, если нужно)

## Проверка
1. Зайти на `https://домен/admin`
2. Ввести логин/пароль
3. Убедиться, что редирект на Dashboard работает
4. Проверить доступ к `/admin/guests` и другим ресурсам
