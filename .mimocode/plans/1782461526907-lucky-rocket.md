# Auto-generate favicon from event poster

## Концепция
При загрузке постера (PNG с прозрачным фоном) автоматически генерируются: favicon 32x32 и apple-touch-icon 180x180. Каждое мероприятие имеет свою иконку.

## Изменения

### 1. Установка Intervention Image
```bash
composer require intervention/image
```

### 2. Контроллер: `app/Http/Controllers/FaviconController.php`
Два метода:
- `show(Event $event)` — `/favicon/{event:slug}.png` → 32x32 PNG
- `appleTouch(Event $event)` — `/apple-touch-icon/{event:slug}.png` → 180x180 PNG
- Если poster_image нет → отдаёт стандартный `public/favicon.png`

Генерация on-the-fly из poster_image с прозрачным фоном (preserve transparency).

### 3. Маршруты: `routes/web.php`
```php
Route::get('/favicon/{event:slug}.png', [FaviconController::class, 'show'])->name('event.favicon');
Route::get('/apple-touch-icon/{event:slug}.png', [FaviconController::class, 'appleTouch'])->name('event.apple-touch-icon');
```

### 4. Observer: `app/Observers/EventObserver.php`
При сохранении Event — если poster_image изменился, генерирует и сохраняет в `storage/app/public/events/favicons/{id}-32.png` и `{id}-180.png`.

### 5. Blade-шаблоны — обновить `<head>`
- `home.blade.php` → favicon + apple-touch-icon из активного события
- `registration.blade.php` → аналогично
- `event/show.blade.php` → добавить favicon + apple-touch-icon + manifest

### 6. Observer регистрация: `app/Providers/AppServiceProvider.php`

## Верификация
1. Загрузить постер → в `storage/app/public/events/favicons/` появятся файлы
2. `/` → favicon =缩小ённый постер активного события
3. `/events/{slug}` → favicon =缩小ённый постер этого события
4. `php artisan test` → OK
