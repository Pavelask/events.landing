# Импорт ответов из Яндекс Форм через API

## Цель
Восстановить `YandexFormsApi.php` и создать Artisan-команду для массового импорта ответов из Яндекс Форм в таблицу `participants` (обезличенные данные: `answer_id`, `event_id`, `checkin_token`, `status`).

## Изменяемые файлы

### 1. `app/Services/YandexFormsApi.php` — восстановить из git history
Коммит `d854c96` содержит исходную версию. Методы:
- `getAnswers(string $formId, array $filters = [])` — GET `/forms/{formId}/answers` с пагинацией
- `getAnswer(string $formId, string $answerId)` — GET `/forms/{formId}/answers/{answerId}`
- Токен из `config('services.yandex.token')`

### 2. `app/Console/Commands/ImportFromYandexForms.php` — создать
```
php artisan import:yandex-forms {formId} {--event-id=}
```
Логика:
1. Получить form_id из аргумента
2. Определить event_id: из опции `--event-id` или найти событие по `yandex_form_id`
3. Цикл с пагинацией: `GET /forms/{formId}/answers?page=N&pageSize=100`
4. Для каждого ответа:
   - Извлечь `answer_id` из `$answer['id']`
   - Проверить дубликат: `Participant::where('answer_id', $answerId)->exists()`
   - Если нет — создать запись с `event_id`, `answer_id`, `checkin_token` (random 40), `status=registered`
5. Вывести статистику: импортировано / пропущено / ошибки

### 3. `config/services.php` — проверить `yandex.token`
Уже существует.

## Порядок выполнения

1. Восстановить `YandexFormsApi.php` из git (`git show d854c96:app/Services/YandexFormsApi.php`)
2. Создать `app/Console/Commands/ImportFromYandexForms.php`
3. Закоммитить и запушить
4. На сервере: `sudo -u apache git pull && sudo -u apache php artisan import:yandex-forms 6a14036190fa7bee223a8ecf`

## Проверка
- Команда выводит статистику: импортировано / пропущено / ошибки
- В `admin/participants` появляются участники с `answer_id`
- Повторный запуск не создаёт дубликатов
